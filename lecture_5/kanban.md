# Канбан

- основана на Toyota, развиваемой с 50-ых годов

- Бережливое производство TPS (Toyota Production System)

    - сокращение расходов на логистику
    - исключение производства лишних деталей
    - синхронизация разных участников производства

- канбан доска 
    - система карточек для связи потоков между отделами

- Канбан - адаптация TPS к процессу конструирования программного обеспечения
- Основа: визуализация управления командой и контроля процесса
- Цели:
    - синхронизация процессов
    - балансировка нагрузки
    - получение результата в оговоренные сроки

## Артефакты
- бэклог 
    - открыт для изменений
- задачи 
    - задачи связаны с проектов
    - задача имеет состояние
    - могут быть другие состояния кроме (todo, in progress, done)
- каждая задача имеет приоритет
    - срочный
    - с фикс датой
    - стандартный
    - нематериальный

- Приоритет задачи может меняться 

## Канбан-доска
- основной инструмент (элемент управления)
- приоритет задач меняется путем перемещения карточак в столбцах доски
- состояние задачи фиксируется перемещением в соответсвующий столбец 
- на одной доске могут быть задачи нескольких проектов
- основной принцип эффективности 
    - прекращайте начинать, начните заканчивать
- на отдельные столбцы канбан-доски могут быть наложены ограничения
- для отдельных учатсников могут установлены ограничения

## Коммуникации
- канбан митинг = scrum (ежедневно)
- обновление бэклога
    - еженедельно
    - до 30 минут
    - команда
    
- встреча с клиентом
    - 30 минут
    - команда и заказчик

- обзор рисков
    - ежемесячно
    - команда
    - обсуждение прошлых неудач, устранение в будущем

- обзор операций
    - ежемесячно
    - оценка и поиск способов общего повышения эффективности всех команд и подразделений

- обзор стратегий
    - ежеквартально
    - ключевые члены команды и руководство
    - оценка деятельности всей компании, выявление масштабных проблем 

## Отличие от scrum 
- нет спринтов
- канбан доска позволяет делать ограничения
- для задач применяется класс обслуживания
- канбан более гибкий

канбан годится к процессам у которых: 
    - не специфицировано время окончания
    - новые задачи появляются в любое время
- идеальное место - поддержка!!
